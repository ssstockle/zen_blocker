Privacy Policy — Zen Blocker

Last updated: 2025-12-29

Summary

Zen Blocker only stores data necessary for its local operation: the list of blocked sites, attempt counters, and user preferences. We do not collect personal identifiable information, health data, financial data, personal communications, or page content.

What data is stored

  • Domain/patterns added by the user (block list).
  • Attempt counters and block settings.
  • Extension preferences and options.

Where data is stored and transmission

Data is stored in chrome.storage inside the browser:

  • chrome.storage.local: data is stored only on the device and is not transmitted to external servers.
  • chrome.storage.sync (if the user enables it): data is synced to the user’s Google account to replicate settings across devices; it is not sent to the developer's servers or third parties.

No collection or transmission of browsing history

The extension only inspects the top-level URL of navigations to determine whether to block a page. This check runs locally and the extension does not log or transmit browsing history to third parties.

Remote code

No remote JavaScript is executed. All scripts are packaged with the extension and the code does not use eval, new Function, or insert remote <script> tags.

Retention and control

Data is retained until the user deletes it via the extension UI, clears the extension data, or uninstalls Zen Blocker.

Sharing and sale

We do not sell or share user data with third parties.
